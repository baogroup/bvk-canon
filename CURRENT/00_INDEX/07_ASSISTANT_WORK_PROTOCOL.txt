Назначение:
Этот файл закрепляет обязательный рабочий протокол для ассистента при чтении, восстановлении контекста и правке BVK Canon.

Главный принцип:
Ассистент обязан работать не по памяти разговора и не по сокращённым входным точкам, а по живому репозиторию как по источнику истины.

Обязательный порядок восстановления контекста:
1. Сначала открыть полный assistant index.
2. Затем сверить живой CURRENT/00_INDEX/CHANGELOG.txt.
3. Затем открыть целевой файл, с которым идёт работа.
4. Затем сверить связанные документы через LINK_MAP.txt и смысловые связи раздела.
5. Только после этого предлагать правки, новые файлы или changelog-записи.

Правило assistant index:
Полный assistant index является основной рабочей картой канона.
Recovery Scan является только короткой точкой входа и не должен использоваться как единственный источник восстановления контекста для сложной или длинной работы.

Правило changelog:
Номер новой записи CHANGE берётся только из живого CURRENT/00_INDEX/CHANGELOG.txt.
Ассистенту запрещено угадывать номер по памяти, по старому ответу или по устаревшей assistant-странице.

Правило существования файла:
Перед созданием нового файла ассистент обязан проверить полный assistant index и убедиться, что нужный файл уже не существует под тем же или близким названием.

Правило уровней доступа:
Ассистент обязан определять уровень доступа по типу документа, а не по инерции.
- PUBLIC используется для канонических документов мира, законов, моделей, официальных принципов и хронологии мира
- SERVICE используется для внутренних редакционных, организационных, планировочных, narrative и служебных файлов

Правило сервисных документов:
Контент-планы, article matrix, редакционные карты, news-layer scaffolding, narrative planning, внутренние шаблоны и рабочие протоколы по умолчанию относятся к SERVICE, если специально не утверждено иное.

Правило связности:
Ассистент не должен править файл изолированно.
Минимум для сверки:
- сам целевой файл
- CHANGELOG.txt
- MASTER_INDEX.txt
- LINK_MAP.txt
- ближайшие связанные документы по смыслу

Правило после правки:
После каждой значимой правки ассистент должен:
- обновить шапку файла
- добавить корректную запись в CHANGELOG.txt
- проверить, не нужно ли обновить MASTER_INDEX.txt или LINK_MAP.txt
- учесть, должен ли файл быть виден в PUBLIC_READ или только в assistant-слое

Запрещено:
- работать только по Recovery Scan при сложной задаче
- угадывать номер CHANGE
- угадывать уровень доступа
- создавать файл вслепую без проверки индекса
- игнорировать уже существующие связанные документы

Вывод:
Ассистент работает с каноном не по памяти диалога, а по формальному протоколу repo-first, где полный assistant index и живой changelog имеют обязательный приоритет.
