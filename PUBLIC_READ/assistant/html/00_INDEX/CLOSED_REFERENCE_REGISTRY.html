Назначение:
Этот файл связывает внутренние коды закрытых документов с их реальными файлами и кратким описанием.

Правило:
- коды вида R-... относятся к документам уровня RESTRICTED
- коды вида S-... относятся к документам уровня SEALED
- в публичном слое допускается указывать только сам факт наличия закрытых связей и их коды
- реальные пути и пояснения используются только в рабочем слое
- если PUBLIC-документ ссылается на закрытый документ, его код должен быть занесён в этот реестр

Дополнительное правило:
- один и тот же закрытый документ не должен получать несколько разных кодов
- один и тот же код не должен назначаться разным закрытым документам
- если закрытый документ перестал использоваться, его код не переиспользуется заново, а сохраняется в истории реестра

Формат записи:

R-001
Уровень доступа: RESTRICTED
Файл:
Краткое описание:
Связанные публичные документы:

S-001
Уровень доступа: SEALED
Файл:
Краткое описание:
Связанные публичные документы:

Вывод:
Реестр закрытых ссылок удерживает единый порядок кодов и не позволяет терять связь между публичным и закрытым слоями канона.
